# All Personas — Hybrid Matrix Content Logic
## Value + Decision in Every Matrix Cell

## Purpose
This file applies the **hybrid matrix rule** to **all personas**.

Every matrix cell must show:

```md
[Title]

Value
[most relevant KPI / number]

Decision
[brief decision to take]
```

The matrix should not show:
- abstract decision-model labels
- long explanations
- answer paragraphs
- separate “meaning” text on the cell face

The matrix is a **decision surface with data attached**.

---

## Global matrix rule

### Rows = Emotion
1. Cautious
2. Strategic
3. Decisive
4. Analytical

### Columns = Persona-specific perspectives
Each persona has 4 perspectives.
Those 4 perspectives stay fixed for that persona.

### Each cell must contain
1. **Title** — short action-oriented label
2. **Value** — the most relevant KPI / number for that emotion × perspective
3. **Decision** — the brief decision that follows from that value in that state

---

# Universal cell-writing rule

For every persona and every cell, construct content in this order:

## Step 1 — identify the active perspective
What kind of lens is this column using?

Examples:
- self / personal exposure
- finance / value
- delivery / execution
- trust / people
- governance / ethics
- customer / market
- supply / energy

## Step 2 — identify the active emotion
How is the persona deciding right now?

- **Cautious** → reduce downside, avoid avoidable harm
- **Strategic** → maximize long-term outcome, preserve optionality
- **Decisive** → act quickly, remove blockers, create movement
- **Analytical** → compare evidence, justify decision quality

## Step 3 — choose the right value
The value should be:
- numeric if available
- KPI if numeric value is not available
- threshold signal if that is the true driver

## Step 4 — generate the decision
The decision should be a short instruction shaped by:
- persona
- perspective
- emotion
- scenario pressure
- active decision model

---

# Emotion-to-decision logic

## Cautious
Cell decision should sound like:
- protect downside
- do not overcommit
- stay inside safe limits
- avoid trust damage
- prevent irreversible loss

## Strategic
Cell decision should sound like:
- preserve optionality
- back the long-term outcome
- keep the coalition together
- choose the path that ages well
- protect structural advantage

## Decisive
Cell decision should sound like:
- act now
- remove the main blocker
- escalate immediately
- contain the hit fast
- move inside clear limits

## Analytical
Cell decision should sound like:
- compare evidence
- test the trade-off
- justify before committing
- choose the strongest supported path
- quantify downside before action

# Execution rule by decision model

The decision model selected underneath should influence the **decision wording**, not the visual structure.

## If primary model = Prospect Theory
Decision wording should emphasize:
- downside
- loss avoidance
- avoiding an irreversible bad outcome

## If primary model = Bounded Rationality
Decision wording should emphasize:
- good-enough action
- practical next move
- reduced complexity

## If primary model = Dual-Process
Decision wording should balance:
- intuition check
- rational confirmation
- “act, but verify”

## If primary model = Recognition-Primed
Decision wording should emphasize:
- pattern-match
- move on the most familiar workable response
- speed with experience

## If primary model = OODA
Decision wording should emphasize:
- orient fast
- act before momentum is lost
- out-cycle the problem

## If primary model = Expected Utility
Decision wording should emphasize:
- best expected outcome
- weighted trade-off
- calculated value preservation

## If primary model = COM-B / behavior design support
Decision wording should emphasize:
- what is possible now
- remove friction
- enable action capability
